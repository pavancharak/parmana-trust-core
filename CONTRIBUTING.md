\# Contributing to Parmana Trust Core



Thank you for your interest in contributing.



\---



\## Project Mission



Parmana Trust Core provides infrastructure for:



```text

Authorization

&#x20;     ↓

Intent

&#x20;     ↓

Execution

```



The project focuses on execution trust, verifiable authorization, trust chains, trust anchors, trust roots, and federated trust.



\---



\## Development Principles



Contributions should preserve:



\* Determinism

\* Verifiability

\* Traceability

\* Independent Verification

\* Human-Defined Authority



\---



\## Development Workflow



1\. Fork repository

2\. Create feature branch

3\. Implement changes

4\. Add tests

5\. Update documentation

6\. Submit pull request



\---



\## Documentation Requirements



Architectural changes must include documentation updates.



Relevant locations:



```text

docs/architecture/

docs/reference/

docs/guides/

docs/phases/

```



\---



\## Cryptography Changes



Changes affecting:



\* Attestations

\* Trust Anchors

\* Trust Roots

\* Verification

\* Key Rotation



should include:



\* Test coverage

\* Example artifacts

\* Documentation updates



\---



\## Code Quality



All contributions should:



\* Build successfully

\* Pass tests

\* Maintain backward compatibility where practical

\* Preserve trust chain integrity



\---



\## Project Principle



Organizations define authority.



Organizations define policy.



Parmana verifies compliance before execution.
