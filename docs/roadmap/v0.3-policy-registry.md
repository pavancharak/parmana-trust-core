\# Parmana v0.3 Policy Registry Roadmap



\## Objective



Move Parmana from:



```text

Trust Core

```



to:



```text

Authority Verification Platform

```



by introducing:



\* Task Registry

\* Policy Registry

\* Schema Registry

\* Signal Registry



These registries become the organizational source of truth for authority.



\---



\# v0.3 Deliverables



\## Registry Layer



\### Task Registry



Purpose:



Define action categories.



Examples:



```text

payment.release

vendor.create

refund.approve

trade.execute

account.close

```



Endpoints:



```text

POST /tasks

GET /tasks

GET /tasks/:id

PUT /tasks/:id

DELETE /tasks/:id

```



Database Table:



```sql

tasks

```



Fields:



```text

task\_id

name

description

active

created\_at

updated\_at

```

\# Authority Evaluation Engine



\## Endpoint



```text

POST /evaluate

```



Input:



```json

{

&#x20; "taskId": "payment.release",

&#x20; "signals": {

&#x20;   "managerApproved": true,

&#x20;   "kycVerified": true,

&#x20;   "amount": 15000

&#x20; }

}

```



Processing:



```text

Task Lookup

&#x20;    ↓

Policy Lookup

&#x20;    ↓

Schema Lookup

&#x20;    ↓

Signal Validation

&#x20;    ↓

Policy Evaluation

```



Output:



```json

{

&#x20; "decision": "approved"

}

```



Possible Decisions:



```text

approved

denied

escalated

```

\---



\# Canonical Statement



Organizations decide what to trust.



Trusted signals may originate from humans, software systems, banking systems, rule engines, or AI systems.



Parmana evaluates those trusted signals against policy before execution.



Humans define authority.



Parmana makes authority verifiable and enforceable before execution.
