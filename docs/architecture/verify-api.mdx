# Verify API Specification

## Status

Planned

Version:

```text
v1
```

Endpoint:

```http
POST /verify
```

---

## Purpose

The Verify API determines whether an attestation can be trusted.

Evaluation answers:

```text
What was decided?
```

Attestation answers:

```text
What was signed?
```

Verification answers:

```text
Can that signed evidence be trusted?
```

---

## Runtime Position

```text
POST /evaluate
        ↓
Decision
        ↓
POST /attest
        ↓
Attestation
        ↓
POST /verify
        ↓
Verification Result
```

Verification is the trust-validation stage of the Authority Runtime.

---

## Responsibility

The Verify API:

```text
Validates Signatures

Validates Trust Profiles

Validates Threshold Rules

Validates Trust Anchors

Produces Verification Results
```

The Verify API does not:

```text
Evaluate Policies

Create Decisions

Execute Actions

Create Attestations
```

Examples:

```text

enterprise

regulated

```

```text
Accepted Algorithms

Required Signatures

Threshold Rules

Trust Anchors
```

---

## Threshold Verification

Future profiles may require:

```text
1 of 1

2 of 3

3 of 5

5 of 7
```

Example:

```text
3 Signatures Present
       ↓
2 Required
       ↓
Verification Success
```

---

## Trust Anchor Validation

Verification resolves trust anchors.

Examples:

```text
Public Keys

Certificate Chains

Federated Registries

Trust Stores
```

Verification succeeds only if signatures chain to trusted anchors.

---

## The Trust Boundary

Evaluation determines authority.

Verification determines trust.

Model:

```text
Authority
      ↓
Decision

Trust
      ↓
Verification
```

These are separate concerns.

---

## Canonical Verification Statement

Verification does not determine what is allowed.

Verification determines whether evidence of authority can be trusted.

Policies define authority.

Attestations create trust evidence.

Verification validates that trust evidence.

Parmana makes authority verifiable and enforceable before execution.
