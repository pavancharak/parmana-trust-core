\# Trust Anchor Model



\## Purpose



The Trust Anchor is the cryptographic root of trust for Parmana.



Its responsibility is to establish verifiable trust in authorization artifacts generated by the Parmana Execution Trust Chain.



The Trust Anchor does not:



```text

Evaluate Policy

Make Decisions

Execute Actions

```



The Trust Anchor only:



```text

Signs

Verifies

Establishes Trust

```



for execution authorization artifacts.

\---



\# Design Goals



\## Root Of Trust



All execution authorization artifacts derive trust from a single root authority.



```text

Parmana Root Key

&#x20;       ↓

Execution Token

&#x20;       ↓

Execution Authorization

```



\---



\## Independent Verification



A third party must be able to verify Parmana-issued authorization artifacts without calling Parmana.



Verification requires:



```text

Execution Token

Public Key

```



only.



\---



\## Persistence



Trust must survive:



```text

Server Restarts

Deployments

Horizontal Scaling

```



A restart must not invalidate previously issued tokens.



\---



\## Rotation



Trust anchors must be replaceable.



Future root versions must preserve trust continuity.



\---



\# Current Implementation



\## Algorithm



```text

Ed25519

```



Selected because:



```text

Fast Verification

Small Signatures

Modern Security

Simple Deployment

```



\---



\## Root Key Pair



Current structure:



```text

packages/trust-anchor/keys



root-private.pem

root-public.pem

```



\### root-private.pem



Used for:



```text

Execution Token Signing

```



Must never be distributed.



\---



\### root-public.pem



Used for:



```text

Execution Token Verification

External Verification

Audit Verification

```



May be distributed publicly.

\# Trust Properties



The Trust Anchor guarantees:



\## Authenticity



```text

Parmana signed this artifact.

```



\---



\## Integrity



```text

Artifact contents were not modified.

```



\---



\## Non-Repudiation



```text

Authorization can be proven later.

```



\---



\## Restart Safety



```text

Restart

&#x20;  ↓

Same Root Key

&#x20;  ↓

Previously Issued Tokens Remain Valid

```



This property was achieved through persistent PEM-backed root keys.
