\# INV-200: Execution Must Match Authorized Intent



\## Status



```text

ACTIVE

```



\## Category



```text

Execution Trust

```



\## Severity



```text

CRITICAL

```



\## Overview



INV-200 ensures that execution remains cryptographically bound to authorized intent.



An execution may only occur if the execution payload exactly matches the intent that was authorized, attested, verified, and transformed into an Execution Trust Token.



This invariant prevents unauthorized modification of execution after approval.



\---



\## Statement



```text

Execution payload must exactly match

the authorized intent.

```



Formally:



```text

SHA256(executionPayload)

&#x20;         ==

&#x20;     intentHash

```



Where:



```text

intentHash

```



is the hash of the authorized execution intent.



\---



\## Purpose



Traditional systems often authorize an action but do not verify that execution remained within approved boundaries.



This creates a trust gap:



```text

Approval

&#x20;   ↓

Execution

```



Execution may be modified after approval without detection.



INV-200 closes this gap by requiring execution to remain cryptographically identical to the authorized intent.



\---



\## Trust Principle



Parmana establishes trust between authority and execution.



```text

Authority defines what is allowed.



Intent defines what is supposed to happen.



Parmana makes execution verifiable.

```



INV-200 enforces the Intent → Execution boundary.



\---



\## Enforcement Point



INV-200 is enforced during execution authorization.



```text

Authority Decision

&#x20;       ↓

Intent

&#x20;       ↓

Attestation

&#x20;       ↓

Verification Receipt

&#x20;       ↓

Execution Trust Token

&#x20;       ↓

INV-200 Check

&#x20;       ↓

Execution

```



Execution is blocked if the invariant fails.



\---



\## Verification Algorithm



Expected value:



```text

expectedHash

&#x20;   =

token.intentHash

```



Actual value:



```text

actualHash

&#x20;   =

SHA256(executionPayload)

```



Validation:



```text

actualHash

&#x20;     ==

expectedHash

```



Result:



```text

true  → execution allowed



false → execution denied

```

Authorized Intent:



```json

&#x20; "amount": 1000,

&#x20; "recipient": "merchant-123"

}

```



Generated:



```text

intentHash =

405ff5dccd32dbb7c36d5b2778916da4b41bff04e170f71ecc84cf9aebcc3715

```



Execution Payload:



```json

{

&#x20; "amount": 1000,

&#x20; "currency": "INR",

&#x20; "recipient": "merchant-123"

}

```



Generated:



```text

actualHash =

405ff5dccd32dbb7c36d5b2778916da4b41bff04e170f71ecc84cf9aebcc3715

```



Result:



```text

INV-200 PASSED

```



Execution proceeds.



\---



\## Violation Example



Authorized Intent:



```json

{

&#x20; "amount": 1000,

&#x20; "currency": "INR",

&#x20; "recipient": "merchant-123"

}

```



Execution Payload:



```json

{

&#x20; "amount": 1900,

&#x20; "currency": "INR",

&#x20; "recipient": "merchant-123"

}

```



Expected:



```text

405ff5dccd32dbb7c36d5b2778916da4b41bff04e170f71ecc84cf9aebcc3715

```



Actual:



```text

a69f539775f6d72b8b9fbf42759c7f736092655451a4bfaf35118a873c031f19

```



Result:



```text

INV-200 FAILED

```



Execution denied.



\---



\## Failure Response



When INV-200 fails:



```json

{

&#x20; "error": "INV-200 violation: execution payload does not match authorized intent",

&#x20; "expectedHash": "<intentHash>",

&#x20; "actualHash": "<executionHash>"

}

```



No execution record is created.



No execution authorization is granted.
